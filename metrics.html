Evaluation Metrics in Machine Learning

A complete reference to the metrics used to measure model performance — across classification, regression, and clustering tasks.

Classification Regression Clustering

When building machine learning models, we need to know how well they perform. Evaluation metrics let us measure effectiveness and choose the best model for our goals. Selecting the right metric is just as important as selecting the right algorithm.

SECTION 1 Classification Metrics

Classification = predicting a discrete category (Yes/No, Spam/Not Spam, Disease/Healthy).

Accuracy

Proportion of all predictions that were correct

The most basic metric. It tells us what percentage of all predictions the model got right. Simple and intuitive, but can be deceptive with imbalanced datasets.

Accuracy = Number of Correct Predictions / Total Predictions
⚠ Warning: In a dataset with 90% Class A and 10% Class B, a model always predicting A scores 90% accuracy — but never detects Class B at all.

Precision

Of all "positive" predictions, how many were actually positive?

Precision focuses on the quality of positive predictions. Use when false positives are costly — like diagnosing a healthy person as sick, or blocking a legitimate email.

Precision = TP / (TP + FP)
TP = True Positives  |  FP = False Positives (predicted positive, actually negative)
catch
every
case 3

Recall (Sensitivity)

Of all actual positives, how many did we correctly find?

Recall measures coverage. Use when missing a positive case is costly — like missing a cancer diagnosis or a security threat. High recall = fewer cases slipping through.

Recall = TP / (TP + FN)
FN = False Negatives (predicted negative, but actually positive)
best of
both 4

F1 Score

Harmonic mean of Precision and Recall — the balanced metric

When you need both precision AND recall to be high, use F1. It combines them into a single number. Range: 0 (worst) to 1 (best). Especially useful with imbalanced classes.

F1 = 2 × (Precision × Recall) / (Precision + Recall)
✓ Tip: F1 is the go-to metric when your dataset has class imbalance (one class is much rarer than the other).
5

Logarithmic Loss (Log Loss)

Measures how confident and correct the model's probability outputs are

Log Loss penalizes the model for being confidently wrong. A model that says "99% sure this is positive" when it's actually negative gets a very high penalty. Lower Log Loss = better.

Log Loss = −(1/N) × Σᵢ Σⱼ yᵢⱼ · log(pᵢⱼ)
yᵢⱼ = actual class (0 or 1)  |  pᵢⱼ = predicted probability  |  Goal: minimize Log Loss
6

AUC / ROC Curve

Overall ability to distinguish between positive and negative classes

The ROC curve plots True Positive Rate vs False Positive Rate at every possible threshold. AUC (Area Under the Curve) gives a single number: probability that the model ranks a random positive example higher than a random negative one.

TPR = TP / (TP + FN)
True Positive Rate (Sensitivity / Recall)
FPR = FP / (FP + TN)
False Positive Rate (1 − Specificity)

AUC interpretation scale

< 0.5Worse than random
0.5Random guessing
0.7–0.9Good model
1.0Perfect model

SECTION 2 Regression Metrics

Regression = predicting a continuous numeric value (price, temperature, sales volume).

simple &
robust 1

Mean Absolute Error (MAE)

Average of the absolute differences between predicted and actual values

Easy to interpret — in the same units as the target variable. Treats all errors equally. Less sensitive to outliers than MSE.

MAE = (1/N) × Σ |yⱼ − ŷⱼ|
punishes
big
errors 2

Mean Squared Error (MSE)

Average of squared differences — penalizes large errors more

Squaring amplifies large errors. Useful when big mistakes are especially costly. Output is in squared units (harder to interpret directly).

MSE = (1/N) × Σ (yⱼ − ŷⱼ)²
3

Root Mean Squared Error (RMSE)

Square root of MSE — back in the original units

Like MSE but interpretable. Best of both worlds: penalizes large errors, yet expressed in the target's original units (dollars, degrees, etc.).

RMSE = √[ (1/N) × Σ (yⱼ − ŷⱼ)² ]
wide-
range
values 4

Root Mean Squared Log Error (RMSLE)

For targets that span many orders of magnitude

Applied to log-transformed values. Penalizes underestimations more than overestimations. Ideal for predicting prices, populations, or any wide-range quantity.

RMSLE = √[ (1/N) × Σ (log(yⱼ+1) − log(ŷⱼ+1))² ]
goodness
of fit 5

R² (R-Squared)

Proportion of variance in the target explained by the model

Tells us how much better the model is vs. simply predicting the mean every time. R²=1 is perfect; R²=0 means the model explains nothing.

R² = 1 − [ Σ(yⱼ − ŷⱼ)² / Σ(yⱼ − ȳ)² ]
✓ R² close to 1 → good model. Close to 0 → poor fit. Can go negative if the model is worse than just predicting the mean.

SECTION 3 Clustering Metrics

Clustering = unsupervised grouping of similar data points. No ground-truth labels exist.

1

Silhouette Score

How well does each point fit its cluster vs. the nearest neighbor cluster?

Measures cohesion (closeness within cluster) and separation (distance to other clusters). Range: −1 to +1.

Silhouette = (b − a) / max(a, b) a = avg distance to same-cluster points b = avg distance to nearest other cluster
Score ≈ +1 → well clustered, point is in the right group
Score ≈ −1 → point likely in the wrong cluster
2

Davies-Bouldin Index

Average similarity between each cluster and its most similar neighbor

Measures how separated and compact the clusters are. Unlike Silhouette, lower is better. Goal: minimize this index.

DB = (1/N) × Σ max_{i≠j} [ (σᵢ + σⱼ) / d(cᵢ, cⱼ) ] σᵢ = avg distance of points in cluster i from centroid d(cᵢ, cⱼ) = distance between cluster centroids i and j
✓ Lower DB index = clusters are more compact and better separated.

QUICK REF When to Use Which Metric

Metric Type Use when… Avoid when…
Accuracy Classification Classes are balanced Dataset is imbalanced
Precision Classification False alarms are costly (spam filter) Missing cases is the bigger concern
Recall Classification Missing positives is costly (cancer detection) False alarms are very costly
F1 Score Classification Need balance of precision & recall; imbalanced data One metric clearly matters more
AUC/ROC Classification Evaluating across all thresholds; binary tasks Multi-class tasks (use macro-F1)
MAE Regression Want interpretable errors; outliers present Large errors must be penalized heavily
RMSE Regression Large errors are unacceptable; interpretable units needed Many outliers skewing results
R² Regression Comparing models; assessing variance explained Absolute error size matters most
Silhouette Clustering Choosing number of clusters (k) Very large datasets (computationally expensive)
Davies-Bouldin Clustering Comparing clustering algorithms Non-convex cluster shapes
